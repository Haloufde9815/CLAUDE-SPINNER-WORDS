## Vulnerability Reporting Protocol

In accordance with established technical and professional standards for security disclosure, security-related observations associated with this project are documented through formal scholarly channels.

To document a security concern, communication is facilitated with the project curator:
  - **Curator**: [Amey Thakur](https://github.com/Amey-Thakur)
  - **Method**: Reports are submitted via the repository’s [GitHub Issues](https://github.com/Amey-Thakur/CLAUDE-SPINNER-WORDS/issues) interface to formally record security-related findings.

Submissions include:
  1. A precise and technically accurate description of the identified issue.
  2. Demonstrable steps or technical evidence sufficient to contextualize the finding.
  3. An explanation of the issue’s relevance within the sovereign scope of the registry.

## Implementation Context: Claude Spinner Words Registry

Claude Spinner Words is a high-fidelity, autonomous technical simulator and registry for the Claude Code CLI environment. It features a sovereign configuration management architecture that enables deterministic control over terminal state indicators. The system utilizes a zero-dependency automation pipeline for `settings.json` manipulation, coupled with a hardware-accelerated workstation emulator deployed via GitHub Actions.

-   **Scope Limitation**: This policy applies exclusively to the documentation, source code, and logic manifests contained within this repository and does not extend to the Claude Code CLI runtime, browser execution environments, or third-party terminal configurations.

## Technical Integrity Statement

This repository is preserved as a fixed engineering project. Security-related submissions are recorded for documentation and contextual reference and do not imply active monitoring, response obligations, or subsequent modification of the repository.
